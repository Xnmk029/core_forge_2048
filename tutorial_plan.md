# CORE FORGE 教学关卡设计 Plan

> **实施状态（2026-07-22）**：4 关教学全部实装（v1.1，2747 行）。Toast 引擎 15 种触发器、训练假人守卫（HP 6 / 无行动）、主菜单教学状态机（进度显示 / 跳过 / 二级菜单锁定）、SAVE 持久化、自动关卡递进。冒烟测试 60 项断言全过（含教学 1 通关 → 自动进入教学 2）。

## 0. 设计参照：Balatro 教学结构

Balatro 用 8 局渐进式引导跑（Tutorial Run）完成全机制教学：

| 特征 | Balatro 做法 | CORE FORGE 移植 |
|------|-------------|----------------|
| 局数 | 8 局，每局引入 1–2 个机制 | 4 关，每关引入 1 个机制层 |
| 长度 | 缩短 Ante 目标（2–3 级即胜） | 缩短至 1–2 层，守卫 HP 大幅削减 |
| 引导方式 | 上下文 tooltip（操作时弹出） | 事件触发 toast（合并/受伤/灾厄出现时弹出） |
| 解锁 | 顺序解锁，必须逐局完成 | SAVE 持久化进度，逐关解锁 |
| 通关奖励 | 解锁完整游戏 + 卡组 | 解锁自由下潜 + 炉主档案全功能 |
| 可跳过 | 可跳过（但推荐完成） | 可跳过（主菜单"跳过教学"按钮） |
| 棋盘/牌组控制 | 预设牌组保证教学牌型出现 | 预设棋盘种子 + 关闭随机生成，保证教学局面 |

核心原则：**每一关只教一件事，用"做中学"代替文字说明**。玩家不是读说明书，而是在引导下亲手完成操作，toast 只在关键节点出现一句话。

## 1. 四关教学设计

### 第 1 关：锻压入门（2048 基础 + 角落策略）

**教学目标**：滑动、合并、合并 = 伤害、大数堆角落

**场景设置**：
- 守卫：训练假人（TRAINING-DUMMY）—— HP 30，无攻击模式（pattern 为空数组），纯沙包
- 棋盘：预设开局（非随机），关闭随机生成（spawnRandom 跳过）
  - 初始布局：`[2,2,.,. / .,.,.,. / .,.,4,. / .,.,.,4]` —— 按 ← 即可合并一对 2，自然形成"向左下角聚集"的局面
- 能量：0（本关不涉及技能）
- 角色：固定炽（无角色选择）

**引导脚本**（事件触发 toast，每关 5–7 条）：

| # | 触发条件 | Toast 内容 | 备注 |
|---|---------|-----------|------|
| 1 | 关卡开始 | 按 方向键 或 WASD 滑动核心，相同数值碰撞会合并翻倍 | 首次显示，不自动消失，等第一次移动后消 |
| 2 | 首次合并发生 | 合并产生伤害！看守卫血条——数字飞过去了 | 配合飘字动画 |
| 3 | 累计 3 次合并 | 技巧：把最大的核心固定在角落，沿一边向外排列 | 角落策略核心提示 |
| 4 | 首次出现 16 号核心 | 16 号核心已锻成——继续合并，击穿训练假人 | 正反馈 |
| 5 | 守卫 HP < 30% | 假人快撑不住了——最后一击！ | 收尾引导 |
| 6 | 守卫死亡 | 教学完成！合并是唯一的进攻手段，记住角落原则 | 结算 |

**完成条件**：击败训练假人
**解锁**：第 2 关

### 第 2 关：守卫与意图（战斗循环 + 血量管理）

**教学目标**：守卫会反击、意图预报怎么读、完整度 = 生命线

**场景设置**：
- 守卫：熔渣傀儡（正常模型，但 speed 5 → 更慢，给玩家反应时间）
- 棋盘：标准开局（2 枚随机核心），恢复随机生成
- 能量：0
- 角色：固定炽

**引导脚本**：

| # | 触发条件 | Toast 内容 |
|---|---------|-----------|
| 1 | 关卡开始 | 这次守卫会反击——看右侧「意图预报」，红色条是倒计时 |
| 2 | 首次受到攻击 | 你被击中了！完整度下降——归零即炉毁 |
| 3 | 首次合并 ≥16 | 合并越高阶的核心，伤害越大——尽快锻出大数 |
| 4 | 守卫 HP < 50% | 守卫过半了——注意自己的完整度，别同归于尽 |
| 5 | 守卫死亡 | 教学完成！读意图、抢输出、保完整度——这是生存三角 |

**完成条件**：击败熔渣傀儡
**解锁**：第 3 关

---

### 第 3 关：灾厄与技能（炸弹 + 清除 + 超频）

**教学目标**：灾厄方块的反制、能量经济、两个主动技能

**场景设置**：
- 守卫：爆破蠕虫（正常模型，会产炸弹）
- 棋盘：标准开局
- 能量：60（足够释放两次技能，降低首次使用门槛）
- 角色：固定炽

**引导脚本**：

| # | 触发条件 | Toast 内容 |
|---|---------|-----------|
| 1 | 首枚炸弹出现 | 蠕虫产下了炸弹核心——看红色倒计时，归零会爆炸伤你 |
| 2 | 炸弹出现后 2 秒 | 两种拆法：把它和同数值核心合并，或按 1 用「清除」直接销毁 |
| 3 | 首次使用清除（按 1 + 点击） | 清除消耗 30 能量——能量靠合并积攒，花在刀刃上 |
| 4 | 能量 ≥ 45 且未用过超频 | 能量够了——按 2 试试「超频」，接下来 3 步合并伤害 ×2.5 |
| 5 | 首次使用超频 | 超频期间全力合并——这是爆发窗口 |
| 6 | 守卫死亡 | 教学完成！灾厄要反制、能量要规划、技能要抓时机 |

**完成条件**：击败爆破蠕虫
**解锁**：第 4 关

---

### 第 4 关：遗物与流派（构筑方向 + 学派共鸣）

**教学目标**：遗物选择、学派颜色识别、3 件共鸣、构筑规划意识

**场景设置**：
- F1 守卫：熔渣傀儡（快速击杀，进入遗物选择）
- F1 遗物选项：**脚本化**——固定 3 件热力学派遗物（过载线圈 / 链式引信 / 点火核心），保证玩家看到"同学派三选一"
- F2 守卫：爆破蠕虫（正常）
- F2 遗物选项：正常随机（但池里热力遗物权重提高，大概率再出热力）
- 能量：30
- 角色：固定炽（热力罗盘，与脚本遗物配合演示共鸣）

**引导脚本**：

| # | 触发条件 | Toast 内容 |
|---|---------|-----------|
| 1 | F1 遗物模态弹出 | 选择一件遗物——注意卡片顶部的颜色，那是学派标识 |
| 2 | 拾取首件遗物后 | 你拿到了热力遗物——集齐 3 件同学派触发「共鸣」，获得强力被动 |
| 3 | F2 开始 | 第二层——继续拿热力遗物凑共鸣，还是混拿其他学派？这是构筑抉择 |
| 4 | 共鸣触发（3 件热力） | 热力共鸣激活！合并伤害 +20%——这就是流派构筑的力量 |
| 5 | F2 守卫死亡 | 教学全部完成！正式下潜中，5 位炉主各对应一个学派——去档案里选你的路线 |

**完成条件**：击败 F2 守卫
**解锁**：教学完成标记（SAVE.tutorialDone = true）

## 2. 提示系统（Toast Engine）

### 2.1 数据结构

```js
const TUTORIALS = [
  {
    id: 1, name: '锻压入门', guardian: 'dummy',
    board: [2,2,0,0, 0,0,0,0, 0,0,4,0, 0,0,0,4],  // 预设棋盘
    noSpawn: true, energy: 0, charId: 'chi',
    hints: [
      { trigger: 'start', text: '按 方向键 或 WASD 滑动核心...', sticky: true },
      { trigger: 'firstMerge', text: '合并产生伤害！...' },
      { trigger: 'mergeCount>=3', text: '技巧：把最大的核心固定在角落...' },
      { trigger: 'maxTile>=16', text: '16 号核心已锻成...' },
      { trigger: 'guardianHp<0.3', text: '假人快撑不住了...' },
      { trigger: 'guardianDead', text: '教学完成！...' },
    ]
  },
  // ... 2, 3, 4
];
```

### 2.2 触发器类型

| 触发器 | 检测点 | 参数 |
|--------|--------|------|
| `start` | startRun 后 | — |
| `firstMove` | doMove 首次成功 | — |
| `firstMerge` | onMerge 首次 | — |
| `mergeCount>=N` | onMerge 计数 | N |
| `maxTile>=V` | onMerge 后 maxTile 检查 | V |
| `firstHit` | hurtPlayer 首次 | — |
| `firstBomb` | executeAction bomb 首次 | — |
| `skillUsed` | tryPurge/tryOverdrive 首次 | skillId |
| `energy>=N` | gainEnergy 后 | N |
| `guardianHp<X` | damageGuardian 后 hp/maxHp | X (0–1) |
| `guardianDead` | guardianDefeated | — |
| `relicOffer` | offerRelics 弹出 | — |
| `relicPicked` | pickRelic 后 | — |
| `affinityTriggered` | schoolCount 达到 3 | school |
| `floorStart` | spawnGuardian | floor |

### 2.3 Toast UI

- 位置：棋盘上方居中（#combo-stamp 同区域，但样式不同）
- 样式：半透明深色底 + 学派色左边框 + 白色文字 + 进度点（● ● ○ ○ ○）
- 行为：
  - `sticky: false`（默认）：显示 4 秒后自动淡出
  - `sticky: true`：等到下一个触发器事件才消失（用于"按方向键"这种需要等操作的提示）
  - 同时只显示 1 条，新 toast 替换旧 toast
- 动画：从上方滑入（translateY -20px → 0），淡出（opacity 1 → 0）

### 2.4 与现有系统的集成

- `Run.startRun(seed, charId, tutorialId)` —— tutorialId 非空时进入教学模式
- `Board.spawnRandom` —— `if(S.tutorial && S.tutorialDef.noSpawn) return null;`
- `spawnGuardian` —— 教学关使用 TUTORIALS[n].guardian 覆盖 guardianKeyFor
- `onMerge / hurtPlayer / executeAction / damageGuardian / guardianDefeated / offerRelics / pickRelic` —— 各插入 `checkTutorialHints(eventName)` 调用
- `SAVE` —— `sv.tutorial: { completed: [], skipped: false }` 持久化
- 主菜单 —— 教学未完成时，"开始下潜"按钮变为"继续教学（第 N 关）"；旁边小字"跳过教学"

## 3. 训练假人守卫定义

```js
// 加入 GUARDIANS 表（仅教学使用）
dummy: {
  code: 'TRAINING-DUMMY', name: '训练假人',
  hpMult: 0.35,  // 40×1^1.35×0.35 = 14 → 取整约 14 HP，2-3 次合并即死
  dmgMult: 0, speed: 99,  // 不攻击
  minFloor: 0, pattern: [],  // 空模式
  desc: '不会反击的锻练靶。用它熟悉锻压节奏。'
}
```

Sprite 渲染：dummy 类型画一个静态灰色方块（不旋转、不脉冲），与正式守卫区分。

## 4. 主菜单集成

### 教学进度状态机

```
sv.tutorial.completed = []     → 主菜单显示"开始教学"（替代"开始下潜"）
sv.tutorial.completed = [1]    → 显示"继续教学 · 第 2 关：守卫与意图"
sv.tutorial.completed = [1,2]  → 显示"继续教学 · 第 3 关：灾厄与技能"
sv.tutorial.completed = [1,2,3]→ 显示"继续教学 · 第 4 关：遗物与流派"
sv.tutorial.completed = [1,2,3,4] → 教学完成，"开始下潜"恢复，炉主档案全功能
sv.tutorial.skipped = true     → 跳过教学，直接进入完整游戏
```

### 主菜单按钮逻辑

```
if (!sv.tutorial.skipped && sv.tutorial.completed.length < 4) {
  主按钮 = "继续教学 · 第 N 关"  → startTutorial(N)
  副按钮 = "跳过教学"（小字，需 confirm）
  炉主档案 / 炉体升级 按钮可见但锁定（tooltip: 完成教学后解锁）
} else {
  主按钮 = "开始下潜"（正常流程）
  炉主档案 / 炉体升级 正常可用
}
```

## 5. 教学关与正式局的差异

| 维度 | 教学关 | 正式局 |
|------|--------|--------|
| 角色 | 固定炽 | 玩家选择 |
| 棋盘 | 第 1 关预设，其余标准 | 标准随机 |
| 随机生成 | 第 1 关关闭 | 开启 |
| 守卫 | 脚本化（假人/减速傀儡/蠕虫） | 按层确定性选取 |
| 遗物选项 | 第 4 关 F1 脚本化 | 随机 + 角色池过滤 |
| 炉渣产出 | 无（教学关不计入经济） | 正常 |
| 解锁判定 | 不触发角色解锁 | 正常 |
| 炉温 | 固定 0 | 玩家选择 |
| 升级树 | 不生效 | 生效 |

## 7. 验收标准

- 第 1 关：新玩家 60 秒内完成（按提示滑动 → 合并 → 击杀假人），toast 逐条触发无遗漏
- 第 2 关：玩家首次被击中时 toast 准确弹出，意图预报可读
- 第 3 关：炸弹出现后 2 秒内弹出反制提示，清除/超频各至少使用 1 次
- 第 4 关：遗物模态弹出时 toast 解释学派颜色，共鸣触发时 toast 庆祝
- 教学进度跨刷新持久化（reload 后从上次关卡继续）
- 跳过教学后主菜单恢复正常，不再出现教学入口
- 教学关不产出炉渣、不触发角色解锁
- 冒烟测试：CF.startTutorial(1) → 自动按方向键 → 击杀 → 断言 SAVE.tutorial.completed 包含 1
- 全程 node --check 通过、单测全绿、冒烟零控制台错误
